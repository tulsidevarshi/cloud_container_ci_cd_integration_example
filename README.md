🗺️ Architectural Summary of the Hurdles
Before jumping into the commands, here is exactly what was happening behind the scenes to cause the issues:
•	The Docker Architecture Hurdle: Building an image on an Apple Silicon (M1/M2/M3) Mac compiles it for arm64. Azure servers run on amd64. We fixed this by appending the --platform linux/amd64 flag during compilations.
•	The CLI Validation Block: The az functionapp CLI module blocked us from configuring container properties directly because of strict internal runtime validators. We bypassed this by manipulating the underlying hosting layer with the generic az webapp command module.
•	The Empty Metadata Runtime (Blue Screen): The official Azure Functions base image spins up a specialized Job Host Process. If it scans the container directory and finds 0 functions loaded, it gives up and falls back to a generic blue placeholder page. We fixed this by nesting your function.json properly and explicitly setting the application path parameters.
